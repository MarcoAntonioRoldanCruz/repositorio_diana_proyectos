Instructivo y Práctica Guiada para Resolución de Conflictos

Utilizando Visual Studio Code y GitHub

Introducción

En el desarrollo colaborativo de software, es común enfrentar conflictos al trabajar en el mismo archivo. Este instructivo te guiará en la resolución de conflictos utilizando Visual Studio Code (VS Code) y GitHub.

Requisitos

  • Tener instalado Git.
  • Tener instalado Visual Studio Code.
  • Tener una cuenta de GitHub.

Pasos Preparatorios

  1. Configurar Git: 
    git config --global user.name "Tu Nombre"
git config --global user.email "tuemail@example.com"
  2. Clonar un Repositorio: 
    git clone https://github.com/tu_usuario/tu_repositorio.git
cd tu_repositorio

Instructivo Paso a Paso

Paso 1: Trabajar en el mismo archivo

Imagina que tú y tu compañera están trabajando en el mismo archivo en la rama main.

Paso 2: Realizar Cambios

  1. Tu compañera realiza cambios en el archivo y realiza un commit: 
    echo "Cambios de la compañera" >> archivo.txt
git add archivo.txt
git commit -m "Añadido cambios de la compañera"
  2. Tú realizas cambios en el mismo archivo antes de actualizar tu copia local: 
    echo "Tus cambios" >> archivo.txt
git add archivo.txt
git commit -m "Añadido tus cambios"

Paso 3: Obtener Últimos Cambios y Generar un Conflicto

  1. Intenta obtener los últimos cambios desde GitHub: 
    git pull origin main

    Aquí, Git generará un conflicto en archivo.txt.

Paso 4: Resolver Conflictos en Visual Studio Code

  1. Abre el repositorio en Visual Studio Code: 
    code .
  2. Ve al archivo con conflictos (archivo.txt). Verás marcadores de conflicto: 
    <<<<<<< HEAD
Tus cambios
=======
Cambios de la compañera
>>>>>>> origin/main
  3. Selecciona cómo resolver el conflicto (aceptar tus cambios, los de tu compañera, ambos, o editar manualmente).
    • Haz clic en "Accept Current Change" para aceptar tus cambios.
    • Haz clic en "Accept Incoming Change" para aceptar los cambios de tu compañera.
    • Haz clic en "Accept Both Changes" para mantener ambos cambios.
    • O edita manualmente el archivo para resolver el conflicto.
  4. Después de resolver el conflicto, guarda el archivo.
  5. Realiza el commit de la resolución del conflicto: 
    git add archivo.txt
git commit -m "Resuelto conflicto en archivo.txt"

Paso 5: Subir Cambios a GitHub

  1. Sube los cambios a GitHub: 
    git push origin main

Práctica Guiada

Ejercicio: Resolver un Conflicto en la Misma Rama

  1. Clona el repositorio de práctica: 
    git clone https://github.com/ejemplo/ejemplo-conflicto.git
cd ejemplo-conflicto
  2. Ambos trabajan en el mismo archivo:
    • Tu compañera realiza cambios y hace un commit: 
      echo "Cambios de la compañera" >> archivo_conflicto.txt
git add archivo_conflicto.txt
git commit -m "Añadido cambios de la compañera"
    • Tú realizas cambios y haces un commit: 
      echo "Tus cambios" >> archivo_conflicto.txt
git add archivo_conflicto.txt
git commit -m "Añadido tus cambios"
  3. Intenta obtener los últimos cambios desde GitHub: 
    git pull origin main

    Se generará un conflicto en archivo_conflicto.txt.

  4. Abre VS Code y resuelve el conflicto en archivo_conflicto.txt.
  5. Guarda los cambios y realiza el commit de la resolución: 
    git add archivo_conflicto.txt
git commit -m "Resuelto conflicto en archivo_conflicto.txt"
  6. Sube los cambios a GitHub: 
    git push origin main

Conclusión

Con este instructivo y práctica guiada, ahora sabes cómo resolver conflictos de manera eficiente utilizando Visual Studio Code y GitHub. La clave es mantener la calma, entender los cambios realizados y tomar decisiones informadas sobre cómo fusionar el trabajo de diferentes colaboradores en la misma rama.
